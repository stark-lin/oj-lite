# Hint 控制规则

建议把 Hint 分成三类，不要直接写出核心代码。

## Simple Hint

可以提示使用哪个结构。

```text
Hint：先判断条件，再用一个变量记录结果。
```

不要写成：

```text
Hint：如果 score >= 60，就把 passed 设为 true。
```

---

## Normal Hint

可以提示解题步骤，但不要给完整条件。

```text
Hint：先计算当前状态，再根据状态判断属于哪一类。
```

不要写成：

```text
Hint：先算 hp - damage，如果 hp > 0 就活着，否则失败。
```

---

## Challenge Hint

只提示拆题方向。

```text
Hint：把问题拆成“枚举方案、判断是否合法、更新最优答案”三步。
```

不要写成：

```text
Hint：用两层循环枚举 a 和 b，如果 a * priceA + b * priceB <= budget，再比较 a + b。
```

# 补充说明

从 Week 02 开始，题目难度按“本周练习—旧知结合—综合挑战”递进。

Simple 题只检查本周核心概念，避免同时引入多个旧知识点。

Normal 题允许结合前几周已经学过的变量、判断、循环、计数、累加或数组操作，但题目目标仍然围绕本周主题。

Challenge 题必须体现综合性，通常需要同时使用 2 到 4 个算法点，例如变量更新、条件判断、循环枚举、计数、累加、数组遍历、查找或排序。

Hint 只提供解题方向，不直接给出关键判断条件、循环范围、更新公式或完整代码。
